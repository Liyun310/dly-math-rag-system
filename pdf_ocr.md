# 高中数学试卷 PDF 本地 OCR 解析脚本需求 (Prompt for AI Coder)

## 1. 项目背景与目标
我目前正在开发一个高中数学 RAG 系统。第一步需要将一批包含复杂排版、几何图形和海量数学公式（LaTeX）的中文试卷 PDF，精准转换为包含 LaTeX 公式的标准 Markdown 文件。

**当前任务：**
请帮我编写一个基于 Python 的自动化批量 OCR 脚本 `pdf_to_markdown.py`。
推荐使用目前在科学公式提取上表现最优的开源库（如 **MinerU (magic-pdf)** 或 **Marker**）。请你根据这两个库目前的最佳实践，选择一个为其编写调用代码。

---

## 2. 运行环境与硬件加速要求
- **硬件环境：** 本脚本将运行在搭载 Apple Silicon (M4 芯片) 的 Mac 上。
- **强制要求：** 请务必在代码中（如 PyTorch 设备设置、或具体 OCR 库的推理配置中）显式开启对 `mps` (Metal Performance Shaders) 的支持，以利用统一内存架构进行推理加速。如果选用库的默认行为不支持直接传 `device='mps'`，请在注释中说明如何在 Mac 环境下最优运行。

---

## 3. 核心功能与目录结构要求

### 3.1 目录结构规划
脚本需要能自动处理以下输入输出结构：
- `data/raw_pdfs/`：存放待处理的原始 `.pdf` 试卷和答案文件。
- `data/md_outputs/`：存放 OCR 生成的 `.md` 文件及附带的图片目录（如果有截图保存功能）。

### 3.2 批量处理逻辑
- 使用 `pathlib` 扫描 `data/raw_pdfs/` 下所有的 `.pdf` 文件。
- 引入 `tqdm` 进度条，显示整体处理进度（例如：处理第 3/50 份试卷）。
- 对每一个 PDF 文件，调用 OCR 引擎将其解析为 Markdown 文本。
- 将生成的 Markdown 文本保存到 `data/md_outputs/` 下，文件名与原 PDF 保持一致（例如 `试卷A.pdf` -> `试卷A.md`）。

### 3.3 健壮性与日志记录
- 数学试卷的解析极其消耗内存，且个别损坏的 PDF 可能导致程序崩溃。
- 请在单份 PDF 处理的逻辑外层包裹 `try-except`。如果某份试卷处理失败，打印 Error 日志（包含文件名和错误信息），并跳过继续处理下一份，**绝对不能让整个批处理脚本闪退**。
- 引入 `logging` 模块，将成功、失败和耗时信息记录到本地 `ocr_process.log` 文件中。

---

## 4. 代码输出规范
- 输出完整的 `pdf_to_markdown.py` 代码。
- 在代码头部或末尾，提供运行此脚本所需的依赖安装命令（例如 `pip install marker-pdf torch torchvision...`），并特别注明在 Mac 下安装这些依赖的注意事项。
- 代码必须包含清晰的中文注释，解释 OCR 模型初始化的核心参数（如 batch_size、语言设置等）。
